## Checklist

- [ ] Merge or otherwise bring the upstream `prediction-market-analysis` codebase into this repo while preserving provenance.
- [ ] Update the root `README.md` to explain the upstream relationship and new trading direction.
- [ ] Create the initial `docs/trading/` document set.
- [ ] Add repo structure for `src/exchanges/`, `src/trading/`, `data/`, and `runtime/`.
- [ ] Add `.gitignore` rules for runtime state, local archives, env files, and extracted datasets that must not be committed.
- [ ] Confirm `origin` and `upstream` remotes are correct.
- [ ] Write down local repo boundary rules so this stays separate from the art app.
- [ ] Record first-pass development conventions in `claude.md` and `AGENTS.md`.

## Exit Criteria

- [ ] The repo clearly reads as a trading project, not an empty placeholder.
- [ ] A new engineer can understand the repo purpose and upstream relationship from docs alone.
- [ ] The repo has safe locations for source, docs, data, and runtime artifacts.
